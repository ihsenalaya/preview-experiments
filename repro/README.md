# Reproduction Package

Companion to the paper *Checkpoint-based Database Isolation Reduces
Non-deterministic Test Variance by Two Orders of Magnitude in an Industrial
Kubernetes Setting: A 5-Stack Empirical Study*.

This package contains everything needed to (a) re-execute our analyses against
the raw frozen CSVs we provide, and (b) re-run a subset of the experiments
on a fresh Kubernetes cluster.

---

## TL;DR — re-derive every paper number in 2 minutes

```bash
# 1. Get the artifacts
git clone <repo-url> preview-experiments
cd preview-experiments
git checkout <release-tag>            # e.g. v1.0-tse-submission

# 2. Verify integrity of frozen CSVs
sha256sum -c results/frozen/CHECKSUMS.sha256   # generated by repro/checksums.sh

# 3. Re-derive every analysis table + figure (≈90 s)
python3 -m venv .venv && source .venv/bin/activate
pip install -r analysis/requirements.txt
python3 analysis/build_all.py
python3 analysis/effect_sizes_and_ci.py        # T1 TSE-grade stats
python3 analysis/check_k_consistency.py        # K-invariance audit

# 4. Compare against shipped tables
diff -r results/analysis/tables/ <(git show v1.0-tse-submission:results/analysis/tables/)
```

Expected: zero differences. The `effect_sizes_and_ci.py` script is
deterministic (`SEED=20260517` in `analysis/effect_sizes_and_ci.py`).

## Full re-execution (requires Kubernetes)

### Hardware target

Our results were collected on a 3-node Azure Kubernetes Service cluster
(`Standard_D4s_v3`; 12 vCPU / 48 GiB total). Any cluster with comparable
capacity should reproduce within ±10% on RQ3 wall-clock measurements.

For first-time replication we recommend a **single-node Kind cluster** on a
workstation with ≥16 GiB RAM and 8 vCPU. The harness auto-detects Kind and
adjusts batch sizes accordingly.

```bash
kind create cluster --config repro/kind-config.yaml
helm install preview-operator \
  ../preview/preview-operator/charts/preview-operator \
  --set image.tag=v1.0.45                       # appVersion shipped with the paper
```

### Single-subject sanity (≈ 10 min on Kind)

```bash
SUBJECT=s2-listmonk EXPERIMENT=flakiness \
  python3 -u _run_one_subject.py                # 60 runs × 2 conditions
```

Output: `results/s2-listmonk/flakiness_test_outcomes_<TS>.csv`.
The flakiness rate under iso=True is expected to be exactly 0/60.

### Full 5-subject 4-RQ pipeline (≈ 14 h on the AKS cluster used in the paper)

```bash
./_launch_phase_b_max.sh                         # max-parallel pipeline
./_launch_s5_baseline_full.sh <PID_A> <PID_B>    # baseline mode=migration N=60
python3 scripts/consolidate_results.py
python3 analysis/build_all.py
python3 analysis/effect_sizes_and_ci.py
```

## Stat-pack provenance (T1 TSE-grade hardening)

Every effect size, bootstrap CI, variance test, and TOST equivalence
reported in the paper is regenerated by `analysis/effect_sizes_and_ci.py`
from frozen CSVs only. The script is:

- **Deterministic**: hard-coded `SEED = 20260517` for all bootstrap resamples.
- **Stateless**: no caching, no side effects outside
  `results/analysis/tables/T1_*.{csv,md,tex}`.
- **Read-only on `results/frozen/`**: enforced by `analysis/build_all.py`'s
  `load_frozen()` helper, which is the sole loader.
- **Bootstrap budget**: `B = 10,000` resamples per CI; runtime ≈ 30 s.

## Manifest hash verification

`results/frozen/MANIFEST.json` lists every CSV with its SHA-256.
To validate the bundle has not been tampered with:

```bash
python3 -c "
import json, hashlib, pathlib
m = json.load(open('results/frozen/MANIFEST.json'))
ok = 0
for e in m['entries']:
    p = pathlib.Path('results/frozen') / e['frozen_path']
    h = hashlib.sha256(p.read_bytes()).hexdigest()
    assert h == e['sha256'], f'mismatch: {e[\"frozen_path\"]}'
    ok += 1
print(f'verified {ok} files')
"
```

Expected output: `verified 22 files` (one per `final`-status entry in MANIFEST).
